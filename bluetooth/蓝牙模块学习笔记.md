## 音频格式

无损音频格式

| **音频格式** | 特点                                                     | 优点                                              | 缺点                                   |
| ------------ | -------------------------------------------------------- | ------------------------------------------------- | -------------------------------------- |
| WAV          | 无损音频格式，支持PCM等多种编码，音质高，但文件体积大    | 音质无损，兼容性极佳，几乎所有设备和软件都支持    | 文件体积大，不适合网络传输或大量存储   |
| FLAC         | 无损压缩格式，压缩率约30%-50%，支持元数据标签            | 音质无损，文件体积比WAV小，开放源代码，兼容性较好 | 压缩率不如有损格式高，文件仍较大       |
| ALAC         | 苹果开发的无损压缩格式，类似于FLAC，主要用于苹果生态系统 | 音质无损，文件体积相对较小，支持元数据标签        | 兼容性较差，主要限于苹果设备和iTunes   |
| APE          | 无损压缩格式，压缩率高（50%-70%）                        | 音质无损，文件体积较小                            | 解码速度慢，不适合实时播放，兼容性较差 |

有损音频格式

| **音频格式** | **特点**                                                 | **优点**                                   | **缺点**                                                   |
| ------------ | -------------------------------------------------------- | ------------------------------------------ | ---------------------------------------------------------- |
| MP3          | 最流行的有损压缩格式，通过压缩减小文件体积，音质相对较好 | 文件体积小，适合网络传输和存储，兼容性极佳 | 质有损，低比特率下音质较差                                 |
| AAC          | MP3的继任者，音质和压缩效率更高，支持多声道              | 音质优于MP3，文件体积小，支持多声道音频    | 兼容性不如MP3，部分设备需要特定解码器                      |
| WMA          | 微软开发的有损格式，支持DRM（数字版权管理）              | 音质好，文件体积小，支持版权保护           | 兼容性不如MP3，需Windows Media Player或其他支持WMA的播放器 |
| OGG          | 开放源代码的有损格式，音质与MP3相当                      | 音质好，文件体积小，无专利限制             | 兼容性不如MP3，部分设备需要特定解码器                      |

### 2、架构解析

#### 1. HW层---蓝牙芯片层（controller层）

| RF（RADIO）<br />射频层                                    | 本地蓝牙数据通过射频发送给远端设备，并且通过射频接收来自远端蓝牙设备的数据 |
| ---------------------------------------------------------- | ------------------------------------------------------------ |
| **BB（BASEBAND）<br />基带层**                             | 进行射频信号与数字或语音信号的相互转化，实现基带协议和其它的底层连接规程 |
| **LMP（LINK MANAGER PROTOCOL）<br />链路管理层**           | 负责管理蓝牙设备之间的通信，实现链路的建立、验证、链路配置等操作 |
| **HCI（HOST CONTROLLER INTERFACE）<br />主机控制器接口层** | HCI层在芯片以及协议栈都有，芯片层面的HCI负责把协议栈的数据做处理，转换为芯片内部动作，并且接收到远端的数据，通过HCI上报给协议栈。 |
| **BLE PHY<br />物理层**                                    | 指定低功耗蓝牙所用的无线频段、信号调制解调方式               |
| **BLE LL<br />链路层**                                     | 负责数据的发送和接收，选择什么信道进行通信，识别空中数据包，保证数据完整性 |

#### 2.Transport——数据传输层

此部分在硬件接口（UART/USB/SDIO）实现HOST跟CONTROLLER的交互

| H2   | 基于USB的传输                                                |
| ---- | ------------------------------------------------------------ |
| H4   | 基于UART的传输，最简单的传输方式，只在`HCI raw data`前面加上一个`type` |
| H5   | 基于UART的传输                                               |
| BCSP | 基于UART的传输                                               |
| SDIO | 基于SDIO的传输                                               |

H4需要蓝牙芯片的UART_TX/UART_RX/UART_CTS/UART_RTS/VCC/GND接到MCU；

而H5只需要蓝牙芯片的UART_TX/UART_RX/VCC/GND接到MCU就可以通信。

### 3.Host层---协议层

#### 1.HCI 主机接口层

负责透过Transport把协议栈的数据发送给蓝牙芯片，并且接受来自蓝牙芯片的数据，数据主要分为

HCI COMMAND(HOST->CONTROLLER),

HCI EVENT(HOST<-CONTROLLER),

HCI ACL(HOST<->CONTROLLER),

HCI SCO(这个有点些微差异，因为部分芯片的SCO数据不是透过TRANSPORT直接跟HOST沟通，而是通过特殊的引脚，PCM IN/OUT/SYNC/CLK脚来传输数据)

#### 2.L2CAP---逻辑链路控制与适配协议

将ACL数据分组交换为便于高层应用的数据分组格式，并提供协议复用和服务质量交换等功能。通过协议多路复用、分段重组操作和组概念,向高层提供面向连接的和无连接的数据服务,L2CAP还屏蔽了低层传输协议中的很多特性，使得高层协议应用开发人员可以不必了解基层协议而进行开发

#### 3.SDP---服务发现协议

为应用程序提供了一种方法来发现哪些服务可用，并确定这些可用服务的特征。

#### 4.RFCOMM---串口仿真协议

上层协议蓝牙电话，蓝牙透传SPP等协议都是直接走的RFCOMM

#### 5.OBEX---对象交换协议

蓝牙电话本，蓝牙短信，文件传输等协议都是走的OBEX

#### 6.HFP（Hands-Free）---蓝牙免提协议

一共分为两个角色：AG跟HF

蓝牙耳机跟手机连接，那么手机的角色就是AG，蓝牙耳机的角色是HF

![HFP](D:\嵌入式资料\My_Girant\嵌入式笔记\bluetooth\img\HFP.png)

#### 8.SPP---蓝牙串口协议

#### 9.IAP---苹果的特有协议，分为IAP1/IAP2

#### 10.PBAP---蓝牙电话本访问协议

一共分为两个角色：PCE跟PSE

车载蓝牙跟手机连接，车载蓝牙下载手机的电话本，那么手机的角色就是PSE，车载蓝牙就是PCE

和PBAP一样

#### 12.OPP---对象推送协议

#### 13.AVCTP---音视频控制传输协议

#### 14.AVDTP---音视频分布传输协议

A2DP的底层

#### 15.HID（HUMAN INTERFACE DEVICE）---人机接口协议

HID有很多广泛的用途的，比如蓝牙鼠标，蓝牙键盘，蓝牙自拍杆，蓝牙手柄等，学好HID还是能做很多产品的

#### 16.A2DP---蓝牙音乐协议

分为source段和sink端

蓝牙耳机跟手机连接，手机传输音乐给蓝牙耳机，那么手机就是A2DP source端，蓝牙耳机是A2DP sink端

#### 17.AVRCP---蓝牙音乐控制协议

拿手机跟蓝牙耳机举例（前提是蓝牙耳机有上一首下一首的功能），那么蓝牙耳机就是controller(CT),手机就是target(TG)

#### 18.ATT：蓝牙属性协议

用于发现、读、写对端设备的协议(针对BLE设备),ATT允许设备作为服务端提供拥有关联值的属性集 ，让作为客户端的设备来发现、读、写这些属性；同时服务端能主动通知客户端。

ATT属性分为3个内容

- 属性类型(attribute type)，用UUID的形式来表现
- 属性句柄(attribute handle)，用于标识一个属性
- 属性权限(permissions), 控制是否该Attribute可读、可写、属性值是否通过加密链路发送！

#### 19.GATT：蓝牙通用属性协议

述2框架定义了服务(Server)和服务属性(characteristic)的过程(Procedure)及格式 。
Procedure定义了characteristic的发现、读、写、通知(Notifing)、指示(Indicating)及配置characteristic的广播。

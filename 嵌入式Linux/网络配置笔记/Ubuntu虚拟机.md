# Ubuntu虚拟机

## 1、VMware网络配置

物理机连接到局域网，如网关192.168.1.0

局域网网卡分配给物理机一个IP地址，如192.168.1.11

**Host-only主机模式：**

- 物理机新建虚拟机时也新建了一个虚拟网卡，如192.168.5.0
- 此时虚拟网卡会为每个新建的虚拟机分配地址，如虚拟机1：192.168.5.5，虚拟机2：192.168.5.6
- 同时主机也被虚拟网卡分配了一个内部的IP地址，如192.168.5.1
- 此时在这块虚拟网卡下，主机和虚拟机之间，虚拟机和虚拟机之间就建立了通信网络
- 此模式下虚拟机没办法同外部通信

**NAT模式：**

- 物理机新建虚拟机时建立一块nat地址转换的虚拟网卡

- 虚拟机和虚拟机之间，虚拟机和主机之间通过这块网卡进行通信

- 虚拟同外部通信的方式是通过主机，虚拟机向主机发送请求，主机将请求送到服务器，回复也是如此

  ![NAT模式网络结构图](D:\嵌入式资料\My_Girant\嵌入式笔记\嵌入式Linux\网络配置笔记\NAT模式网络结构图.png)

**桥接模式：**

- 物理机和虚拟机都是同一个网络中的一台设备
- 虚拟机会被分配到和物理机同一个网段的IP地址

## 2、开发调试环境搭建

​	一般开发板需要与PC进行文件互传，例如内核模块，音视频、图片等。文件传输的方法有TCP、串口等方式，但是串口进行文件传送受限与协议一般传输比较慢，所以多数情况下是使用TCP进行传输。

​	如果有路由器情况下，将PC和开发板接到路由器，再配置虚拟机网络配置为桥接模式，这样三端都处于同一地址中，即可完成文件互传。但是，有时候身边会没有路由器，这时候就需要将我们的有线网卡和无线网卡都加以利用起来。我们可以配置使得虚拟机其中的一个虚拟网卡和笔记本电脑的无线网卡在一个网段内，能够连接外网；而另一个虚拟网卡VMnet2、笔记本电脑有线网卡和开发板在另一个网段内，组成一个局域网，达到调试的目的，但这个网段是不能够上网的。

​	**配置主机IP：**

- 将主机的有限网卡地址为静态IP地址，避免IP频繁更换不利于调试；

​	**配置虚拟机IP：**

- 在虚拟机Vmware的网络配置中可以看到VMnet0网卡默认是桥接，将其桥接到主机的无线网卡；
- 创建新的虚拟网卡VMnet2，将其桥接到主机的有线网卡；

## KEIL查看当前项目所用内存

        在keil中编译程序后会输出程序大小，具体解释如下：
        Code：     表示程序所占用 FLASH 的大小（ FLASH）。
        RO-data：即 Read Only-data， 表示程序定义的常量，如 const 类型（ FLASH）。
        RW-data：即 Read Write-data， 表示已被初始化的全局变量（ FLASH,RAM）
        ZI-data：即 Zero Init-data， 表示未被初始化的全局变量(RAM)
在程序初始化的时候，RW-data会从FLASH中拷贝到RAM中,因此在计算flash和ram占用空间时，都得加上RW-data

计算公式：

flash = Code + RO-data + RW-data
ram   = RW-data + ZI-data

以上图为例：
flash=38078+6549+452=45124 bytes       45124 / 1024=44.07 kBytes
ram=452+3276=3728 bytes                       3728 / 1024=3.64 kBytes

## 中断机制

​	Cortex-M3的中断控制器是NVIC，支持嵌套中断。中断优先级分为抢占优先级和子优先级。抢占优先级决定是否能打断当前中断，而子优先级处理同时发生的中断顺序。

​	关键点在于抢占优先级。如果新中断的抢占优先级比当前的高，那么它会打断当前ISR，立即执行。否则，新中断会等待当前ISR完成，或者排队在相同或更低抢占优先级的中断之后。例如，即使两个中断的抢占优先级相同，后来的中断必须等当前的处理完，但如果抢占优先级更高，就会发生嵌套。

​	中断嵌套机制：

- ​	当 **新触发的中断的抢占优先级** 比 **当前正在执行的中断的抢占优先级** **更高**（数值更小）时，会发生 **中断嵌套**：

  ​	当前中断会被 **暂停**（硬件自动保存现场）

  ​	新中断的服务函数会 **立即执行**

  ​	新中断执行完毕后，恢复被暂停的中断继续执行

- 如果新中断的抢占优先级 **相同或更低**（数值更大或相等），则 **不会发生嵌套**：

  ​	新中断会被挂起，等待当前中断执行完毕后才会被响应
